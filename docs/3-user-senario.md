# Team CalTalk 사용자 시나리오 (User Scenarios)

**Version**: v1.0.0 (Based on PRD v2.0.0)  
**Status**: Finalized

## 1. 시나리오 개요
본 문서는 **Team CalTalk**의 주요 기능을 실제 사용자 관점에서 설명하기 위한 시나리오입니다. 각 시나리오는 유스케이스(UC)를 기반으로 구성되었으며, 사용자 경험의 흐름을 정의합니다.

## 2. 전체 사용자 흐름 (User Journey Flow)

Team CalTalk의 전체 사용자 경험은 다음과 같은 선형적/순환적 흐름을 가집니다:

```text
[Start] --> US-01: Onboarding (Registration & Personalization)
                   |
                   v
          [Choice: Team Creation or Join?]
          /                         \
US-02: Create Team (Leader)      US-02: Join via Invite Code (Member)
          \                         /
                   v
        [ Collaboration Loop ] <----------------+
          |                                     |
          +--> US-03: View Calendar & Daily Chat  |
          |                                     |
          +--> US-04: Request/Approve Schedule --+
                   |
                   v
US-05: Continuity (Cross-device Sync) --> [End]
```

1.  **온보딩 (Onboarding)**: 회원가입 → 개인화 설정(언어/테마) [US-01]
2.  **조직화 (Organization)**: 팀 생성 또는 초대 코드를 통한 팀 합류 [US-02]
3.  **협업 루프 (Collaboration Loop)**:
    - **조회**: 캘린더를 통한 팀 일정 파악 [US-03]
    - **소통**: 날짜별 채팅을 통한 실시간 논의 [US-03]
    - **조정**: 일정 변경 요청 및 팀장 승인/반려 [US-04]
4.  **연속성 (Continuity)**: 다른 기기에서 접속 시 기존 설정 및 데이터 자동 동기화 [US-05]

## 3. 주요 시나리오

### US-01: 신규 사용자의 진입과 개인화 설정
*   **사용자**: 김철수 (신규 가입자)
*   **상황**: 평소 다크모드를 선호하고 영어 환경이 익숙한 철수가 서비스를 처음 시작함.
*   **액션**:
    1.  이메일 인증을 통해 회원가입을 완료함.
    2.  설정 메뉴로 진입하여 언어를 'English'로 변경하고 테마를 'Dark Mode'로 전환함.
*   **결과**: 서비스 전체 UI가 영어와 다크모드로 즉시 전환되며, 이 설정값은 서버 DB의 철수 계정 정보에 저장됨.

### US-02: 팀 구성과 협업 시작
*   **사용자**: 이영희 (프로젝트 매니저, 팀장)
*   **상황**: 새로운 'A 프로젝트 팀'을 만들고 팀원들을 초대하려 함.
*   **액션**:
    1.  'Create Team' 버튼을 클릭하여 팀 이름을 입력하고 팀을 생성함.
    2.  시스템에서 생성된 6자리 초대 코드(예: `CT7A9X`)를 복사하여 팀원들에게 메신저로 전달함.
    3.  팀원들은 초대 코드를 입력하여 팀에 합류함.
*   **결과**: 영희는 팀장 권한을 획득하고, 팀원들이 합류함에 따라 팀 캘린더를 공유할 준비가 완료됨.

### US-03: 일정 확인과 실시간 일자별 소통
*   **사용자**: 박민수 (팀원)
*   **상황**: 오늘 오후에 예정된 팀 미팅의 세부 내용을 팀원들과 논의하고 싶음.
*   **액션**:
    1.  캘린더에서 오늘 날짜(2026-05-27)를 클릭함.
    2.  우측에 열린 해당 일자 전용 채팅창에 "오늘 미팅 장소가 어디인가요?"라고 메시지를 보냄.
    3.  접속 중인 팀원들이 실시간으로 답장을 남김.
*   **결과**: 해당 날짜의 일정과 대화가 하나의 맥락으로 연결되어, 나중에 해당 날짜만 클릭해도 대화 내용을 쉽게 확인할 수 있음.

### US-04: 일정 변경 요청 워크플로우
*   **사용자**: 박민수 (팀원) & 이영희 (팀장)
*   **상황**: 민수가 갑작스러운 외부 미팅으로 팀 미팅 시간을 1시간 미루고 싶어함.
*   **액션**:
    1.  민수: 채팅창 내 'Request Change' 버튼을 눌러 미팅 시간을 '14:00'에서 '15:00'로 변경 요청함.
    2.  영희: 실시간 알림을 받고 요청 상세 내용을 확인한 뒤 'Approve' 버튼을 클릭함.
*   **결과**: 캘린더의 미팅 시간이 자동으로 15:00로 업데이트되며, 채팅창에 "요청이 승인되었습니다"라는 시스템 메시지가 표시됨.

### US-05: 기기 변경 시 환경 유지 (Cross-Device Consistency)
*   **사용자**: 김철수 (기존 사용자)
*   **상황**: 사무실 PC에서 다크모드로 사용하던 철수가 외근 중 태블릿으로 처음 로그인함.
*   **액션**:
    1.  태블릿 브라우저를 통해 로그인을 완료함.
*   **결과**: 별도의 설정을 하지 않았음에도 철수가 PC에서 설정했던 'English' 언어와 'Dark Mode' 테마가 그대로 적용되어 표시됨. (서버 사이드 설정 동기화)

## 3. 예외 시나리오 (Edge Cases)
- **잘못된 초대 코드**: 존재하지 않거나 만료된 코드를 입력했을 때 "Invalid code" 메시지 노출.
- **권한 없는 수정 시도**: 팀원이 팀장의 승인 없이 일정을 직접 수정하려고 할 때 UI상에서 차단됨.
- **네트워크 단절**: 채팅 중 연결이 끊겼을 때 재접속 시 메시지 재시도 메커니즘 작동.
