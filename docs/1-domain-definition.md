# Team CalTalk 도메인 정의서 (Domain Definition)

**Version**: v2.0.0  
**Status**: Updated with v2 features and UC structure

## 1. 프로젝트 개요 (Project Overview)
**Team CalTalk**은 팀 단위의 캘린더 일정 관리와 실시간 채팅을 결합하여, 팀원 간의 일정 충돌을 방지하고 소통의 맥락을 유지하는 협업 플랫폼입니다. 일정과 대화의 분리로 인해 발생하는 정보 누락과 소통 비용을 줄이는 것을 목표로 합니다.

## 2. 문제 정의 (Problem Statement)
*   **일정의 파편화**: 개인 캘린더, 메신저 등으로 일정이 흩어져 있어 전체 현황 파악이 어렵고 충돌이 발생함.
*   **맥락(Context) 부재**: 일정 변경 사유나 논의 과정이 파편화되어 사후 추적이 어려움.
*   **개인화 부족**: 사용자의 작업 환경(테마)이나 언어 설정이 유지되지 않아 사용자 경험이 저하됨.

## 3. 비즈니스 목표 (Business Goals)
*   **일정 가시성 확보**: 팀 단위 통합 캘린더를 통한 실시간 공유.
*   **소통의 중앙화**: 일자별 채팅 로그를 통한 일정 논의 맥락 보존.
*   **글로벌 확장성 및 편의성**: 다국어 지원과 서버 저장형 테마 설정을 통한 최적의 사용자 경험 제공.

## 4. 도메인 용어 사전 (Glossary)
| 용어 | 설명 |
| :--- | :--- |
| **팀 (Team)** | 일정과 채팅을 공유하는 최소 단위의 조직. |
| **팀장 (Leader)** | 팀 생성자 혹은 관리 권한을 가진 자. 일정 확정 및 멤버 관리 권한 보유. |
| **팀원 (Member)** | 팀 일정을 조회하고, 채팅을 통해 일정 변경을 요청할 수 있는 사용자. |
| **팀 일정 (Team Schedule)** | 팀 전체 혹은 특정 인원들이 공유하는 캘린더 상의 이벤트. |
| **일자별 채팅 (Daily Chat)** | 특정 날짜에 귀속된 채팅 메시지. 해당 날짜의 일정 논의를 위한 공간. |
| **변경 요청 (Change Request)** | 팀원이 팀장에게 일정 조율을 제안하는 공식적인 프로세스. |
| **요청 상태 (Status)** | 변경 요청의 진행 단계 (대기, 승인, 반려). |
| **초대 코드 (Invite Code)** | 팀에 멤버를 추가하기 위해 생성하는 고유 식별 코드. |
| **테마 (Theme)** | 사용자가 선택한 UI 색상 모드 (Light/Dark). 서버에 저장되어 기기 간 동기화됨. |
| **다국어 (i18n)** | 사용자의 선호에 따라 제공되는 언어 설정 (한국어, 영어 등). |

## 5. 권한 모델 (Permission Model)
*   **공통 (User)**: 팀 생성, 다중 팀 소속, 개인 환경 설정(테마/언어) 변경 가능.
*   **팀장 (Leader)**: 일정 CUD 권한, 변경 요청 승인/반려 권한, 멤버 관리 및 초대 코드 생성 권한.
*   **팀원 (Member)**: 일정 조회(Read-only), 일자별 채팅 참여, 일정 변경 요청 발송 권한.

## 6. 비즈니스 흐름 (UC 기반)
1.  **[UC-01] 인증**: 사용자는 계정을 생성하고 로그인하여 시스템에 접속한다.
2.  **[UC-02] 조직 구성**: 사용자는 팀을 생성(팀장)하거나 초대 코드로 기존 팀에 합류(팀원)한다.
3.  **[UC-03/04] 일정 운영**: 팀장은 전체 일정을 관리하고, 모든 멤버는 실시간으로 이를 공유받는다.
4.  **[UC-05] 맥락 소통**: 특정 날짜의 일정에 대해 해당 일자별 채팅방에서 실시간으로 논의한다.
5.  **[UC-06] 조정 프로세스**: 일정 변경이 필요할 경우 팀원이 요청을 보내고, 팀장이 이를 검토하여 확정한다.
6.  **[UC-07] 개인화**: 사용자는 본인에게 맞는 테마와 언어를 설정하며, 이는 어디서나 동일하게 유지된다.

## 7. 주요 비즈니스 규칙 (Business Rules)
*   **일정 확정권**: 팀의 모든 일정은 팀장의 승인 또는 직접 등록을 통해서만 최종 확정된다.
*   **채팅의 일자 귀속**: 채팅은 방 단위가 아닌 '날짜' 단위로 생성 및 보존된다.
*   **설정의 영속성**: 테마 및 언어 설정은 브라우저 쿠키가 아닌 서버 DB에 저장되어 로그아웃 후 재접속 시에도 유지된다.

## 8. 비기능 요구사항 (Non-functional Requirements)
*   **실시간성**: WebSocket을 이용한 메시지 및 일정 변경 알림의 실시간 전달.
*   **데이터 보존**: 팀 일정 및 채팅 로그는 최소 3년간 영구 보존.
*   **보안**: HTTPS 통신 및 비밀번호 단방향 암호화(Argon2/Bcrypt).
